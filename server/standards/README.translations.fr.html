Guide général

  • Avant de commencer quoique ce soit, lisez
    • les Directives pour les volontaires du site web GNU,
    • les Feuilles de style HTML FSF, http://www.gnu.org/server/fsf-html-style-sheet.html
  • Après lecture de cette page, référez-vous à la liste des traductions en cours, et vérifiez s'il existe déjà une équipe formée pour la langue dans laquelle vous souhaitez traduire. Dans le cas contraire, contactez web-translators@gnu.org, et demandez si vous pouvez créer une nouvelle équipe pour votre langue.

Codes de langue à deux lettres

  • La RFC 1766 décrit les codes de langue qui sont autorisés. L'ISO 639 donne ce codage sur deux lettres dans la deuxième colonne. Les dernières versions de RFC 1766 et ISO 639 devraient s'y trouver.
  • Si l'ISO 639 ne contient pas un code, faites quelques recherches et trouvez le bon code à utiliser. L'ISO 3166 donne les codes pour les pays. Les codes de langues et de pays ne sont pas nécessairement les mêmes. C'est une erreur de supposer qu'un code de pays peut être substitué à un code de langage, sauf si vous l'avez pu le vérifier dans une norme.

Que traduire

  • Les documents devraient être traduits avec leurs pages de référence comme home.html et philosophy/philosophy.html.
  • Ne traduisez aucun document de ces répertoires :
    /software/
    Nous laissons cette section à chaque personne maintenant sa page à jour. Mais si vous souhaitez aider une personne pour traduire ses pages, contactez-la.
    /brave-gnu-world/
    Comment aider à traduire Brave GNU World.
  • En traduisant "GNU's Not Unix," merci d'assurer que la traduction reste récursive. Si une traduction récursive ne peut être conçue, utiliser la forme suivante : "GNU's Not Unix" (GNU N'est pas Unix).

Traduire la page principale (homepage)

  • La page principale traduite doit inclure la liste complète des traductions disponibles via SSI.
    Vous devez insérer un code SSI de ce type : 
    <!--#include file="translations.include" -->
  • Si vous fournissez une nouvelle traduction de la page principale, vous devez mettre à jour le fichier the translations.include. Le fichier est stocké à la racine des répertoires du serveur web de GNU.

Lier d'autres documents

  • Tous les liens sont relatifs à la racine du serveur web, ce qui signifie par exemple qu'ils doivent pointer sur /software/software.html et non juste sur software.html.
  • Link should be made pointing to a translated page if it exists, else made pointing to the English original. You should NOT make links that depend upon content negotiation. One reason is that they will fail on mirror servers that do not do content negotiation.

Traductions des autres langues

  • Le Haut et le Bas de toutes les traductions (en incluant la version anglaise originale) doivent contenir les liens vers les autres traductions du document quand elles existent. Consultez http://www.gnu.org/boilerplate.html pour un exemple de comment faire cela.

Noms de fichier

  • Les traductions doivent être nommées PAGENAME.LG.html où LG is le code de langue à deux lettres. Par exemple, la traduction allemande de home.html doit être nommée  home.de.html.
  • Il existe des liens index.html pointant sur d'autres fichiers other.html. Créez des liens nommés index.LG.html pointant sur la traduction correcte other.LG.html.

Outils pour les traducteurs

  • Un fichier TAGS.LG dans le répertoire ~www/html/ liste tous les fichiers HTML traduits dans la langue LG. Ces fichiers permettent à une personne de chercher ou de créer une requête pour remplacer tous les fichiers HTML pour une langue de code LG. Consulez ces intructions pour plus d'information.

Tenir les traductions à jour

  • Afin que les traductions soient tenues à jour, vous devriez demander aux Responsables Traduction, <web-translators@gnu.org> de vous inscrire à la liste de diffusion diffmon-web. Vous recevrez alors un email pour chaque page qui est modifiée dans le cvs (pou le module www). Le traffic est d'environ 4-5 emails par jour, et plus pendant le week-end.  On préconise que vous regardiez de tels emails et reportiez les modifications environ une fois par semaine pour être sûr que les traductions sont à jour.

Notes finales

  • Si vous traduisez une pages pour www.gnu.org, insérez s'il-vous-plaît le code suivant dans la section <HEAD>...</HEAD> (ou <head> en xhtml). Où web-translators-es@gnu.org est l'adresse d'expédition pour votre équipe de traduction : 
     <link rev="translated" href="mailto:web-translators-es@gnu.org" />
