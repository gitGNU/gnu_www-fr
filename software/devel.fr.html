Introduction

Cette page décrit les services de développement disponibles pour les développeurs GNU sur les machines du Projet GNU. Si vous souhaitez utiliser un de ces services pour le développement d'un paquetage GNU, veuillez contactez l'adresse indiquée pour le service particulier. Pour les détails complets sur les privilèges et responsabilités d'un mainteneur GNU, veuillez voir les Informations pour les mainteneurs des documents GNU . (Les Normes de la Programmation GNU peuvent également être d'intérêt.)

Avec l'abondance d'ordinateurs peu coûteux qui peuvent faire tourner GNU/Linux, ainsi qu'avec des accès Internet toujours plus accessibles, beaucoup de volontaires GNU ont aujourd'hui les installations informatiques dont ils ont besoin. Cependant,ily a encore plus d'avantages à avoir des serveurs où les volontaires peuvent travailler ensemble sans devoir rendre leurs propres machines accessibles aux autres.

Pour cette raison, la Fédération du Logiciel Libre (FSF) encourage les projets de logiciel GNU à utiliser les machines à gnu.org comme plaque tournante. L'utilisation de ces machines bénéfie aussi au Projet GNU indirectement, en augmentant la conscience publique de GNU, et en étendant l'idée de travailler en commun pour le bénéfice de chacun.

CVS et Savannah

Nous fournissons des accès CVS pour beaucoup de paquetage GNU; si vous développez un paquetage GNU et que vous voulez garder le dépôt sur les machines de gnu.org, Savannah offre une façon facile de le créer et manager. Premièrement créez votre compte et ensuite enregistrer votre paquetage GNU. Après quelques heures le dépôt CVS sera créé. Vous pourrez écrire dessus et manager vous-même la liste des personnes qui y auront accès en écriture.

Un certain nombre de projets emploient déjà Savannah dans ce but.

Comptes de Connexion

Nous donnons accès aux machines de GNU aux personnes qui en ont besoin pour travailler sur les logiciels GNU. Avoir un compte est à la fois un privilège et une responsabilité, et il doit être utilisé uniquement pour votre travail sur GNU. Veuillez lire les méthodes d'accès aux machines de GNU afin d'obtenir un compte.

Listes d'Expédition

Nous gérons des listes d'expédition pour les paquetages de logiciels GNU comme nécessaire, incluant à la fois une gestion manuelle et une automatique.

Lorsqu'un paquetage GNU est enregistré sur Savannah, une interface web autorise les développeurs à créer et manager les listes d'expédition attachées à ce paquetage.

Si, pour quelques raisons, enregistrer le paquetage GNU sur Savannah n'est pas impossible ou souhaitable, demandez à new-mailing-list@gnu.org de créer des listes pour vous, ou créer les vous-même si vous êtes sûr de savoir comment faire.

En général, chaque paquetage de logiciel GNU devrait posséder une liste de rapport de bug avec le nom canonique bug-name@gnu.org, ainsi que d'autres alias qui peuvent être utiles. Certains paquetages partagent la liste bug-gnu-utils@gnu.org mais nous encourageons dorénavant aux paquetages d'ouvrir leurs propres listes.

Les paquetages peuvent obtenir d'autres listes pour faire des annonces, demander de l'aide, poster un code source en relation, discuter entre utilisateurs, ou tout ce que le mainteneur du paquetage pense être utile.

Les archives des listes d'expédition, pour les listes gérées automatiquement, sont disponible à http://mail.gnu.org, ainsi qu'à travers le manageur de liste. Les archives pour les listes gérées manuellement sont généralement garder dans /com/archive sur les machines de GNU.

Lorsqu'une liste d'expédition devient assez grande pour la justifier, nous pouvons ouvrir un newsgroup gnu.* avec un accès bi-directionnel à la liste d'expédition.

Serveur Web

Notre serveur web principal est adressé à http://www.gnu.org/, et a des miroirs à travers le monde. Nous aimerions héberger sur ce serveur les pages d'à peu près tous les paquetages de logiciel GNU.

La machine qui sert de serveur pour le site www.gnu.org est séparé du reste des machines de GNU. Les sites sont stockés dans un dépôt CVS sur subversions.gnu.org. Tout mainteneur de paquetage GNU enregistré sur Savannah est garant de l'accés en écriture dans le répertoire /software/package. Pour voir un exemple, sur comment y accéder, examinez la page CVS du paquetage GNU mifluz.

FTP

Nous pouvont aussi fournir un site FTP pour chaque paquetage de lociciel GNU sur http://ftp.gnu.org/, qui possède des miroirs à travers le monde.

Nous utilisons un serveur différent pour les réalisations de test, de sorte que les visiteurs ne les installent pas, pensant qu'elles sont prêtes pour une première utilisation. . Ce serveur est ftp://alpha.gnu.org/.

Le document des Informations pour les mainteneurs GNU a des détails complets sur la procédure d'envoi vers le FTP.
